<Update label="April 23, 2026" description="v2.132.0" tags={["revive-admin"]}>

### New

- **RET Catalog item groups** ‚Äî Level 1 catalog items now have a required `group` field (one of `Interior`, `Exterior`, `MEP`, `Structural`, `Appliances`, `Soft cost`). The Add/Edit Item modal exposes a **Group** dropdown alongside the category name. ([#1752](https://github.com/Revive-Home/revive-admin/pull/1752))
- **Custom item groups in RET** ‚Äî Custom items in the estimate builder now require a **Group** selection in both the Custom Item modal and the review panel. ([#1756](https://github.com/Revive-Home/revive-admin/pull/1756))
- **TBD pricing for custom items** ‚Äî Custom items can now be marked **TBD** when a price isn't yet known. TBD items show `TBD` instead of a currency value in the review panel, side menu, and CSV export, and `value`/`factoredValue` are persisted as `0`. ([#1753](https://github.com/Revive-Home/revive-admin/pull/1753))
- **RMA list ‚Äî Deal Type column** ‚Äî The RMA list view now includes a **Deal Type** column sourced from the associated opportunity's HubSpot dealtype, with a human-readable label mapping (e.g. `Contract Buy-out` ‚Üí `Renovate to sell`). ([#1755](https://github.com/Revive-Home/revive-admin/pull/1755))
- **RMA detail ‚Äî Deal Type and Pipeline Stage** ‚Äî The RMA detail Information panel now displays **Deal Type** and **Pipeline Stage** fields from the opportunity. ([#1755](https://github.com/Revive-Home/revive-admin/pull/1755))
- **RMA `executed_at` timestamp** ‚Äî RMAs now persist an `executed_at` timestamp when they transition to the Executed status. The detail view shows "Executed on M/D/YY at h:mm a". ([#1758](https://github.com/Revive-Home/revive-admin/pull/1758))

### Improved

- **RMA check-run cutoff moved to Friday** ‚Äî The Ready for Check Run and Next Check Run tabs now split on the most recent **Friday 23:59:59** (previously Thursday). The cutoff is evaluated against `executed_at` (falling back to `updated_at`) instead of `created_at`, so RMAs appear in the correct bucket based on when they were executed rather than when they were drafted. Check run totals are now computed from the full tab-filtered set, not just the current page. ([#1758](https://github.com/Revive-Home/revive-admin/pull/1758))
- **Catalog access** ‚Äî `carlo@revive.realestate` and `nehme@revive.realestate` added to the RET Catalog allowed-email list. ([#1756](https://github.com/Revive-Home/revive-admin/pull/1756))
- **Catalog drag-reorder reliability** ‚Äî L1, L2, and L4 catalog reorders now wait for the server update before updating local state, and preserve the `group` field when reordering. ([#1752](https://github.com/Revive-Home/revive-admin/pull/1752))

### Fixed

- **`isTBD` reset on non-custom items** ‚Äî Non-custom RET items no longer retain an `isTBD` flag when edited through the preview panel. ([#1753](https://github.com/Revive-Home/revive-admin/pull/1753))

### Action required

- **Catalog items require a `group`** ‚Äî Existing L1 catalog items without a `group` (or with a group not in the current list) will show an empty **Group** dropdown and must be updated before saving. Keep the admin list in sync with `revive-api`'s `ret-catalog.validation.js` `.valid()` list.
- **Custom items require a `group`** ‚Äî Saving a custom item without selecting a group now fails validation. Existing custom items on in-flight estimates should be re-opened and assigned a group.
- **RMA `dealtype` field is now a label** ‚Äî `useRmas` maps `opportunity.dealtype` through `getDealTypeLabel` before returning it, so consumers see `Renovate to sell`, `Sell 360`, etc. instead of raw HubSpot IDs like `Contract Buy-out` or `Buy It Now`. If you filter on this field client-side, use the label values.
- **RMA check-run filter uses `executed_at`** ‚Äî The Ready and Next Check Run tab split now uses `executed_at || updated_at` and a Friday cutoff. RMAs executed on Friday itself roll into Next Check Run until Saturday.

</Update>

<Update label="April 14, 2025" description="v2.154.0" tags={["revive-api"]}>

### New

- The estimate proposal response now includes a `homeIncrease` field in the `valueOpportunity` object. When present, it contains a formatted currency string (e.g. `"$45,000"`) representing the estimated increase in home value from renovations. ([#1953](https://github.com/Revive-Home/revive-api/pull/1953))

### Improved

- Estimate proposals and opportunity lookups load faster ‚Äî comparables, case studies, HubSpot metadata, and associated records are now fetched concurrently. ([#1953](https://github.com/Revive-Home/revive-api/pull/1953))
- The `trendText` label in the comparison chart now reads `"$X home increase"` instead of `"$X est. profit"` to better describe the value. ([#1953](https://github.com/Revive-Home/revive-api/pull/1953))

### Fixed

No bug fixes in this release.

### Action required

- **`trendText` label changed** ‚Äî The `trendText` field in the `comparisonChart` section of `GET /v1/estimates/{estimateId}/proposal` now uses "home increase" instead of "est. profit". If you parse or match against this string, update your code to expect the new label.

</Update>

<Update label="April 14, 2025" description="v2.153.0" tags={["revive-api"]}>

### New

- The Sell 360 cash offer comparison now subtracts existing mortgage/liens from the cash offer amount, giving homeowners a more accurate side-by-side comparison. ([#1951](https://github.com/Revive-Home/revive-api/pull/1951))

### Improved

- RMA contract generation now resolves the contractor signer from the completed PC contract instead of HubSpot contact associations, ensuring the signer on the RMA matches the signer who executed the PC contract. ([#1951](https://github.com/Revive-Home/revive-api/pull/1951))
- Generating an RMA contract once again requires a signed Purchase Commitment (PC) contract on the associated opportunity (restores the requirement removed in v2.151.0). ([#1951](https://github.com/Revive-Home/revive-api/pull/1951))

### Fixed

- The renovation plan form now correctly saves the **rehab used** amount when updating estimates. Previously, this value could be omitted from the save payload.

### Action required

- **Signed PC contract required for RMA generation** ‚Äî The `POST /v1/rma/:rmaId/contract` endpoint now requires a completed PC contract on the associated opportunity. If no completed PC contract exists, the endpoint returns a `400` error. This reverses the v2.151.0 change. Ensure a PC contract is completed before generating RMA contracts.
- **Sell 360 `comparisonDiff` values will change** ‚Äî The `comparisonDiff` field in `GET /v1/estimates/{estimateId}/proposal` now reflects a lower cash offer baseline (debt is subtracted). If you display or use `comparisonDiff` in calculations, values will be higher for homeowners with existing mortgage balances.

</Update>

<Update label="April 14, 2025" description="v2.152.0" tags={["revive-api"]}>

### New

- Tiered Sell 360 program fee ‚Äî properties with ARV ‚â• $1M now use a 4% rate; below $1M uses 3%. Previously a flat 3% applied to all. ([#1943](https://github.com/Revive-Home/revive-api/pull/1943))
- `debt` and `agentCommission` fields added to the `sell360TransactionSummary.shared` object so clients can display these deductions. ([#1943](https://github.com/Revive-Home/revive-api/pull/1943))
- New `firstCheckAmount` field on opportunity and deal responses, representing the first check amount stored in HubSpot. ([#1943](https://github.com/Revive-Home/revive-api/pull/1943))

### Improved

- Revised holding cost formula: `(initialPayment + renovationCost) √ó 1% √ó holdingMonths` ‚Äî produces lower holding costs for most deals. ([#1943](https://github.com/Revive-Home/revive-api/pull/1943))
- Simplified taxes/insurance/HOA estimate: `asIsPrice √ó 0.16% √ó holdingMonths`. ([#1943](https://github.com/Revive-Home/revive-api/pull/1943))
- Break-even calculation no longer includes holding costs. ([#1943](https://github.com/Revive-Home/revive-api/pull/1943))
- Sell 360 comparison baseline now uses `asIsPrice √ó 90%` instead of a full as-is net calculation. ([#1943](https://github.com/Revive-Home/revive-api/pull/1943))
- Renovation timeline is now budget-based: 1 month per $100K increment, producing more consistent scenario projections. ([#1950](https://github.com/Revive-Home/revive-api/pull/1950))
- Initial payment now sourced directly from the opportunity's `firstCheckAmount` instead of a break-even formula. ([#1950](https://github.com/Revive-Home/revive-api/pull/1950))
- RMA contractor signer now sourced from the completed PC contract instead of HubSpot. ([#1943](https://github.com/Revive-Home/revive-api/pull/1943))

### Fixed

No bug fixes in this release.

### Action required

- **`agentCommissionSold` renamed to `firstTransactionCommission`** ‚Äî Update any client code that reads this field on opportunity and deal responses.
- **`isFirstCheckNegative` removed** ‚Äî This boolean has been removed from `sell360TransactionSummary`. Check `firstCheckNet` directly instead.
- **RMA contract generation requires completed PC contract** ‚Äî `POST /v1/rma/:rmaId/contract` returns `400` if no completed PC contract exists.
- **Sell 360 scenario values will change** ‚Äî All calculated values (initial payment, holding costs, program fees, scenario projections) may differ due to the formula changes above.

</Update>
