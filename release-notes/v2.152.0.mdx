## Highlights

- Sell 360 financial calculations have been updated with a tiered program fee rate, revised holding cost and tax formulas, and a simplified comparison baseline.
- The `agentCommissionSold` field has been renamed to `firstTransactionCommission` across all endpoints, and a new `firstCheckAmount` field is now available on opportunities and deals.
- RMA contract generation now requires a completed PC contract and sources the contractor signer from that contract.

## Changes shipped to production

### revive-api

- **Tiered Sell 360 program fee** — The program fee rate is now based on the after-renovated value (ARV). Properties with an ARV of $1,000,000 or more use a 4% rate; properties below that threshold use 3%. Previously, a flat 3% rate applied to all properties. This affects the initial payment, program fee, and all three scenario projections returned by the estimate proposal endpoint. ([#1943](https://github.com/Revive-Home/revive-api/pull/1943))
- **Revised holding cost formula** — Holding costs are now calculated as `(initialPayment + renovationCost) × 1% × holdingMonths` instead of the previous formula based on the first check base and a 15% annual rate. This produces lower holding costs for most deals. ([#1943](https://github.com/Revive-Home/revive-api/pull/1943))
- **Simplified taxes, insurance, and HOA** — The taxes/insurance/HOA estimate is now `asIsPrice × 0.16% × holdingMonths`, replacing the previous formula that used ARV, an LTV factor, and a 3% annual rate. ([#1943](https://github.com/Revive-Home/revive-api/pull/1943))
- **Updated break-even formula** — The break-even calculation no longer includes holding costs. It now sums the first check base, risk-adjusted renovation cost, sales costs, and program fee. ([#1943](https://github.com/Revive-Home/revive-api/pull/1943))
- **New comparison baseline** — The Sell 360 comparison diff (the difference between Sell 360 proceeds and a traditional sale) now uses `asIsPrice × 90%` as the baseline instead of calculating a full as-is net with sales costs. ([#1943](https://github.com/Revive-Home/revive-api/pull/1943))
- **`debt` and `agentCommission` exposed in proposal response** — The `sell360TransactionSummary.shared` object now includes `debt` and `agentCommission` fields so clients can display these deductions in the transaction breakdown. ([#1943](https://github.com/Revive-Home/revive-api/pull/1943))
- **RMA contractor signer sourced from PC contract** — RMA contract generation now pulls the contractor signer from the completed Professional Contractor (PC) contract on the opportunity instead of looking up an Authorized Signer contact in HubSpot. A completed PC contract is required. ([#1943](https://github.com/Revive-Home/revive-api/pull/1943))

## Breaking changes

### Renamed field: `agentCommissionSold` → `firstTransactionCommission`

The `agentCommissionSold` field has been renamed to `firstTransactionCommission` on opportunity and deal responses. Update any client code that reads this field.

| Endpoint | Old field | New field |
|---|---|---|
| `GET /v1/estimates/{estimateId}` | `agentCommissionSold` | `firstTransactionCommission` |
| Deal responses | `agentCommissionSold` | `firstTransactionCommission` |

### New field: `firstCheckAmount`

A new `firstCheckAmount` field is now returned on opportunity and deal responses. This field represents the first check amount stored in HubSpot.

### RMA contract generation requires a completed PC contract

The `POST /v1/rma/:rmaId/contract` endpoint now requires a completed PC contract on the opportunity. If no completed PC contract exists, the endpoint returns a `400` error with the message: "No completed PC contract found. A completed PC contract is required to create an RMA contract."

This reverses the v2.151.0 change that removed the PC contract requirement. The contractor signer is now sourced from the PC contract's signer list instead of from HubSpot contact associations.

### Sell 360 calculation changes

If you consume the `sell360TransactionSummary` from the estimate proposal endpoint, be aware that all calculated values (initial payment, holding costs, program fees, scenario projections) may differ from previous versions due to the formula changes described above. The `shared` object now also includes two new fields: `debt` and `agentCommission`.
