## Welcome to Revive engineering

This page gets you from zero to a working local environment and your first API call. If you're already set up, jump to the [API Reference](/api-reference/introduction) or [Release Notes](/release-notes/v2.154.0).

---

## Step 1: Get access

Before writing code, make sure you have:

<Steps>
<Step title="GitHub org access">

Request access to [Revive-Home](https://github.com/Revive-Home) from your team lead. You'll need access to at least `revive-api` and the repo you're working on.

</Step>

<Step title="Firebase project access">

Ask your team lead to add you to the Firebase project. You'll need this for authentication (both as a developer and for API tokens).

</Step>

<Step title="Environment variables">

Get the `.env` template from your team lead. Each repo has a `.env.example` — copy it and fill in the values.

</Step>

<Step title="Slack channels">

Join `#engineering`, `#deployments`, and `#api-alerts` in Slack for release notifications and incident updates.

</Step>
</Steps>

---

## Step 2: Run the API locally

```bash
git clone git@github.com:Revive-Home/revive-api.git
cd revive-api
npm install
cp .env.example .env       # fill in keys from your team lead
npm run dev                 # starts on http://localhost:3001
```

Verify it's running:

```bash
curl http://localhost:3001/health
# {"status":"ok","version":"2.152.0"}
```

---

## Step 3: Make your first authenticated call

1. Get a Firebase JWT token (from the dashboard or Firebase Admin SDK)
2. Call an authenticated endpoint:

<CodeGroup>
```bash cURL
curl -sS -H "Authorization: Bearer $TOKEN" \
  http://localhost:3001/v2/users/
```

```javascript JavaScript
const res = await fetch('http://localhost:3001/v2/users/', {
  headers: { Authorization: `Bearer ${token}` }
});
const { data } = await res.json();
console.log(data);
```

```python Python
import requests

resp = requests.get(
    'http://localhost:3001/v2/users/',
    headers={'Authorization': f'Bearer {token}'},
)
print(resp.json())
```
</CodeGroup>

You should see a list of users. If you get a `401`, check that your token is fresh (Firebase JWTs expire after 1 hour).

---

## Step 4: Pick your repo

Depending on what you're working on, clone and run the relevant frontend:

| What you're building | Repo | Command | Port |
|---------------------|------|---------|------|
| Homeowner/agent web app | `revive-dashboard` | `npm run dev` | 3000 |
| Internal admin tools | `revive-admin` | `npm run dev` | 3002 |
| Mobile app | `revive-mobile` | `npm run ios` / `npm run android` | — |

See the [Local development](/development) guide for full setup details and environment variables.
