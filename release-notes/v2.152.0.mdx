### New

- Tiered Sell 360 program fee — properties with ARV ≥ $1M now use a 4% rate; below $1M uses 3%. Previously a flat 3% applied to all. ([#1943](https://github.com/Revive-Home/revive-api/pull/1943))
- `debt` and `agentCommission` fields added to the `sell360TransactionSummary.shared` object so clients can display these deductions. ([#1943](https://github.com/Revive-Home/revive-api/pull/1943))
- New `firstCheckAmount` field on opportunity and deal responses, representing the first check amount stored in HubSpot. ([#1943](https://github.com/Revive-Home/revive-api/pull/1943))

### Improved

- Revised holding cost formula: `(initialPayment + renovationCost) × 1% × holdingMonths` — produces lower holding costs for most deals. ([#1943](https://github.com/Revive-Home/revive-api/pull/1943))
- Simplified taxes/insurance/HOA estimate: `asIsPrice × 0.16% × holdingMonths`. ([#1943](https://github.com/Revive-Home/revive-api/pull/1943))
- Break-even calculation no longer includes holding costs. ([#1943](https://github.com/Revive-Home/revive-api/pull/1943))
- Sell 360 comparison baseline now uses `asIsPrice × 90%` instead of a full as-is net calculation. ([#1943](https://github.com/Revive-Home/revive-api/pull/1943))
- Renovation timeline is now budget-based: 1 month per $100K increment, producing more consistent scenario projections. ([#1950](https://github.com/Revive-Home/revive-api/pull/1950))
- Initial payment now sourced directly from the opportunity's `firstCheckAmount` instead of a break-even formula. ([#1950](https://github.com/Revive-Home/revive-api/pull/1950))
- RMA contractor signer now sourced from the completed PC contract instead of HubSpot. ([#1943](https://github.com/Revive-Home/revive-api/pull/1943))

### Fixed

No bug fixes in this release.

### Action required

- **`agentCommissionSold` renamed to `firstTransactionCommission`** — Update any client code that reads this field on opportunity and deal responses.
- **`isFirstCheckNegative` removed** — This boolean has been removed from `sell360TransactionSummary`. Check `firstCheckNet` directly instead.
- **RMA contract generation requires completed PC contract** — `POST /v1/rma/:rmaId/contract` returns `400` if no completed PC contract exists.
- **Sell 360 scenario values will change** — All calculated values (initial payment, holding costs, program fees, scenario projections) may differ due to the formula changes above.
