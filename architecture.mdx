## System overview

Revive is a multi-service platform connecting homeowners, real estate agents, and contractors through renovation-driven real estate transactions. The tech stack spans a Node.js API, React web apps, a React Native mobile app, and integrations with HubSpot, PandaDoc, Stripe, Firebase, and AI services.

## How services connect

### Authentication flow

1. User signs in via Firebase Auth (dashboard, admin, or mobile)
2. Client receives a Firebase JWT
3. Every API request includes `Authorization: Bearer <jwt>`
4. `revive-api` verifies the JWT via Firebase Admin SDK
5. User role (`admin`, `agent`, `contractor`, `homeowner`) determines access

### Deal lifecycle

```
Agent creates opportunity
  → Property report generated (Vision AI + AVM)
    → Estimate created (Ret Catalog products)
      → Contract sent for e-signature (PandaDoc)
        → RMA created (Renovation Management Agreement)
          → Contractor assigned, draws released
            → Renovation tracked (progress, punch list, media)
              → Case study published (website webhook)
```

### Data flow between integrations

| Integration | Direction | What syncs |
|-------------|-----------|-----------|
| **HubSpot** | Bidirectional | Contacts, companies, deals, deal stages, agent Edge URLs |
| **PandaDoc** | API → PandaDoc → Webhook | Contract creation, signing, completion status |
| **Stripe** | API → Stripe → Webhook | Payment intents, subscription management |
| **Firebase** | Bidirectional | Auth tokens, Firestore documents, Cloud Storage files |
| **Follow Up Boss** | Bidirectional | Contact/lead sync, pipeline stages |
| **House Canary** | API → HC | Market health data, property valuations |

## API versioning

The API uses path-based versioning. Both `/v1` and `/v2` run in parallel.

- **`/v1`** — original endpoints, still fully supported
- **`/v2`** — newer endpoints with improved schemas (deals, users, opportunities, vision-ai, analyze, calendars, payments, referrals)

See the [API changelog](/guides/changelog) for details on what moved to v2 and migration guides.

## Deployment pipeline

```
Developer pushes to main
  → GitHub Actions run tests
    → Heroku auto-deploys revive-api
      → Bot pushes updated openapi.json to revive-api-docs
        → GitHub Action enriches spec + updates banner
          → Mintlify auto-deploys docs
```

Release notes are auto-generated from merged PRs using the `generate-release-notes.mjs` script. See [Release Notes](/release-notes/v2.154.0) for the latest.

## Key directories in revive-api

| Directory | Purpose |
|-----------|---------|
| `src/routes/` | Express route handlers (maps to API endpoints) |
| `src/services/` | Business logic layer |
| `src/models/` | Database models (Sequelize/PostgreSQL) |
| `src/middleware/` | Auth, rate limiting, error handling |
| `src/integrations/` | Third-party API clients (HubSpot, PandaDoc, Stripe, etc.) |
| `src/utils/` | Shared utilities |
| `migrations/` | Database migration files |
