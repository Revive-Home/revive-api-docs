- Introduced “Valuation + Equity Verification” overview with status badges, completion progress, per-document history, and quick navigation. ([#158](https://github.com/Revive-Home/revive-apps/pull/158))
- Added Valuation card for As‑Is, After Renovation, and Renovation Budget. ([#158](https://github.com/Revive-Home/revive-apps/pull/158))
- Added experimental “Sales” tab across multiple opportunity workflows to surface the overview/valuation flow. ([#158](https://github.com/Revive-Home/revive-apps/pull/158))
- Implemented underwriting webhook syncing and surfaced underwriting and equity document status/history on deals. ([#158](https://github.com/Revive-Home/revive-apps/pull/158))
- Multiline AI chat input and a reusable loading spinner for a smoother editor and consistent loading visuals. ([#148](https://github.com/Revive-Home/revive-apps/pull/148))
- "Apply" actions now appear when an AI message is eligible for use. ([#148](https://github.com/Revive-Home/revive-apps/pull/148))
- Users can now toggle between Map and Satellite views in the mapping interface. ([#149](https://github.com/Revive-Home/revive-apps/pull/149))
- Deep links accept latitude/longitude and detailed property fields; Property Details step won’t be auto-skipped. ([#143](https://github.com/Revive-Home/revive-apps/pull/143))
- Photo download (ZIP) export and gated “Download photos” action. ([#143](https://github.com/Revive-Home/revive-apps/pull/143))
- Upload existing media into a deal/gallery. ([#143](https://github.com/Revive-Home/revive-apps/pull/143))
- Out-of-state routing with a dedicated research path and adjusted qualification flow. ([#147](https://github.com/Revive-Home/revive-apps/pull/147))
- Timezone-aware meeting UI so displayed slots and selections align with user timezone. ([#147](https://github.com/Revive-Home/revive-apps/pull/147))
- Heroku 26 stack upgrade. ([#137](https://github.com/Revive-Home/revive-apps/pull/137))
- Tooltip added to the Create RMA request button with contextual display behavior tied to project manager presence. ([#135](https://github.com/Revive-Home/revive-apps/pull/135))
- Backend now exposes a project-manager presence flag and the data model includes an optional project-manager indicator. ([#135](https://github.com/Revive-Home/revive-apps/pull/135))

### Improved

- Enhanced Vision AI search to avoid stale results and keep clear/loading behavior consistent. ([#133](https://github.com/Revive-Home/revive-apps/pull/133))
- Streamlined the Getting Started “research” onboarding flow by removing two intermediate research-specific steps and jumping directly to the remaining shared sequence. ([#166](https://github.com/Revive-Home/revive-apps/pull/166))
- Search is now controlled and more responsive: trimmed input, typing debounced (500ms), Enter/submit and clear run immediately, and closing header search reliably clears the query. ([#152](https://github.com/Revive-Home/revive-apps/pull/152))
- Empty-state handling refined: distinct "no results" vs default "no reports yet" views. ([#152](https://github.com/Revive-Home/revive-apps/pull/152))
- Refined equity-document status normalization and Firestore/HubSpot synchronization logic. ([#158](https://github.com/Revive-Home/revive-apps/pull/158))
- Better sanitization, normalization, and extraction of AI-generated descriptions. ([#148](https://github.com/Revive-Home/revive-apps/pull/148))
- Tighter AI response formatting and instruction handling for more consistent outputs. ([#148](https://github.com/Revive-Home/revive-apps/pull/148))
- Updated the agent request modal’s layout and contact card presentation, including refreshed contact icons and streamlined unauthenticated flow so the modal displays more clearly and consistently. ([#140](https://github.com/Revive-Home/revive-apps/pull/140))
- Code formatting and error logging consolidation for improved maintainability. ([#164](https://github.com/Revive-Home/revive-apps/pull/164))
- Create RMA request button availability updated to require project-manager assignment in addition to existing RMA checks. ([#135](https://github.com/Revive-Home/revive-apps/pull/135))
- Strengthened TypeScript type safety across many components by adding explicit typings for callbacks and selectors. ([#138](https://github.com/Revive-Home/revive-apps/pull/138))
- Cleaned up code by consolidating imports and removing unused error parameters. ([#138](https://github.com/Revive-Home/revive-apps/pull/138))
- No user-facing behavior or public APIs were changed; only compile-time/type and code-quality improvements. ([#138](https://github.com/Revive-Home/revive-apps/pull/138))

### Fixed

- Improved duplicate listing detection with stronger address normalization, including punctuation cleanup, directional/suffix canonicalization, and unit-token removal for more reliable exact matches. ([#167](https://github.com/Revive-Home/revive-apps/pull/167))
- Added a last-resort fuzzy matching approach when exact matching fails, combining street-name similarity with geographic proximity to identify likely duplicates. ([#167](https://github.com/Revive-Home/revive-apps/pull/167))
- Updated deduplication to consistently keep the earliest suitable match among candidates. ([#167](https://github.com/Revive-Home/revive-apps/pull/167))
- Improved baseline/verification behavior by requesting, mapping, and safely deriving the original verified end date when available; uses fallback behavior when values are missing. ([#175](https://github.com/Revive-Home/revive-apps/pull/175))

- Search results protected from out-of-order responses. ([#152](https://github.com/Revive-Home/revive-apps/pull/152))
- Improved required documents modal open/close behavior. ([#158](https://github.com/Revive-Home/revive-apps/pull/158))

- Logs a warning and gracefully handles unrecognized home-type values to prevent incorrect filtering. ([#161](https://github.com/Revive-Home/revive-apps/pull/161))

- Safer avatar/brokerage initials, agent/contact displays, and image lists when names/media are absent; improved address verification to validate responses before use. ([#150](https://github.com/Revive-Home/revive-apps/pull/150))
