## Timestamps

All timestamps are ISO 8601 UTC unless noted otherwise.

| Field | Format | Example |
|-------|--------|---------|
| `createdAt` | ISO 8601 | `"2026-04-15T18:00:00.000Z"` |
| `updatedAt` | ISO 8601 | `"2026-04-15T18:00:00.000Z"` |
| `sentAt` | ISO 8601 | `"2026-04-15T18:00:00.000Z"` |
| `completedAt` | ISO 8601 | `"2026-04-15T18:00:00.000Z"` |
| `signedAt` | ISO 8601 | `"2026-02-15T14:00:00.000Z"` |
| `executed_at` | ISO 8601 | `"2026-04-18T22:15:00.000Z"` |
| `startTime` | Unix ms | `1713100000000` |

<Info>
  `executed_at` is set on an RMA when it transitions to the `executed` status. The Admin RMA list uses this timestamp (falling back to `updated_at`) to decide whether an RMA falls into **Ready for Check Run** or **Next Check Run**, based on the most recent Friday 23:59:59 cutoff.
</Info>

<Warning>
  Some older endpoints use Unix milliseconds (`startTime`, `startDate`) instead of ISO 8601. Check the endpoint reference for the specific format.
</Warning>

## Response envelope

Most endpoints return a consistent envelope:

```json
{
  "status": "success",
  "data": { ... },
  "message": "Optional human-readable message."
}
```

For errors:

```json
{
  "status": "error",
  "message": "What went wrong.",
  "errors": [{ "field": "email", "message": "Email is required." }]
}
```

See [Error handling](/guides/error-handling) for full details.
