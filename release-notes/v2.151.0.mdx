## Highlights

- Generating an RMA contract no longer requires a signed PC contract on the associated opportunity. You only need an owner's rep assigned and an Authorized Signer contact linked to the contractor in HubSpot.

## Changes shipped to production

### revive-admin

- **Simplified RMA contract generation** — Previously, you needed both an assigned owner's rep and a signed Professional Contractor (PC) contract on the opportunity before you could generate an RMA contract. The signed PC requirement has been removed. You can now generate an RMA contract as soon as an owner's rep is assigned. If the owner's rep is missing, the **Generate RMA** button remains disabled with a tooltip explaining what is needed. ([#1746](https://github.com/Revive-Home/revive-admin/pull/1746))

### revive-mobile

No updates in this release.

### revive-api

- **RMA contract signer sourced from HubSpot** — The RMA contract generation endpoint now resolves the contractor signer from the contractor's Authorized Signer contact association in HubSpot instead of pulling it from a signed PC contract. This means you can generate RMA contracts earlier in the deal lifecycle without waiting for a PC contract to be completed. The contractor must have an Authorized Signer contact with a valid email address in HubSpot. ([#1942](https://github.com/Revive-Home/revive-api/pull/1942))

## Fixes

No bug fixes in this release.

## Breaking changes

- **RMA contract generation prerequisites changed** — The `POST /v1/rma/:rmaId/contract` endpoint no longer requires a completed PC contract. Instead, it requires the contractor to have an Authorized Signer contact association in HubSpot with a valid email address. If the Authorized Signer is missing or has no email, the endpoint returns a `400` error. Update any workflows that depend on a signed PC contract being present before generating RMA contracts.
