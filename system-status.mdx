## How it works

| Service         | Endpoint                                            | Check type                    |
| --------------- | --------------------------------------------------- | ----------------------------- |
| HubSpot         | `https://status.hubspot.com/api/v2/status.json`     | Statuspage `status.indicator` |
| Firebase        | `https://status.firebase.google.com/incidents.json` | Ongoing-incident filter       |
| ATTOM (Estated) | `https://api.developer.attomdata.com/home`          | HTTP 200                      |
| RestB           | `https://api-eu.restb.ai/`                          | HTTP 200                      |

The admin API route (`/api/health-status`) runs the checks server-side and caches the response for 60 seconds (`Cache-Control: s-maxage=60`). The page on `/status` re-fetches every minute and re-renders.

<Note>
  Estated's own API (`api.estated.com`) was sunset after ATTOM's 2022
  acquisition. The check points at the ATTOM developer portal instead.
</Note>
