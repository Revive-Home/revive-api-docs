<Update label="April 23, 2026" description="Dashboard Sprint 7" tags={["revive-dashboard"]}>

### New

- **Create brokerage from the dashboard** — Agents can add a new brokerage from **Account Settings → My Profile** without contacting support. The new modal captures name, website, email, phone, address (via autocomplete), logo, and brand colors, and runs a duplicate check against existing records before creating the brokerage. ([#2358](https://github.com/Revive-Home/revive-dashboard/pull/2358))
- **TBD cost labels on custom estimate items** — Renovation estimate custom items can now be flagged as **TBD** when a price has not been finalized. Items marked TBD display a red `TBD` label in place of the total cost in the proposal view, side menu, and products list. ([#2370](https://github.com/Revive-Home/revive-dashboard/pull/2370))
- **Disclaimer text on pre-estimate slides** — Added "Individual results may vary" disclaimers to the **Nearby Case Studies** and **Value Opportunity** sections, and clarified language on the **Sell 360 Scenario Comparison** modal, **Sell 360 Transaction Summary**, and the scenario-comparison PDF. ([#2376](https://github.com/Revive-Home/revive-dashboard/pull/2376))

### Improved

- **Sell 360 proposal finalization** — Updated cash-offer coloring, home-increase copy, row alignment, address trimming, and PDF formatting on the Sell 360 pre-estimate proposal and scenario-comparison modal. ([#2359](https://github.com/Revive-Home/revive-dashboard/pull/2359), [#2364](https://github.com/Revive-Home/revive-dashboard/pull/2364), [#2365](https://github.com/Revive-Home/revive-dashboard/pull/2365), [#2367](https://github.com/Revive-Home/revive-dashboard/pull/2367), [#2368](https://github.com/Revive-Home/revive-dashboard/pull/2368), [#2369](https://github.com/Revive-Home/revive-dashboard/pull/2369))
- **Brokerage create request** — `POST /v1/brokerages/` now accepts optional `email` and `phone` fields. Address fields (`address`, `city`, `state`, `zip`) are now optional at the client level — omit them to create a brokerage without a physical address.

### Fixed

- **Intake flow** — Removed the legacy `refer-renovate` option from the "What can we help with?" step so the intake flow no longer skips screens for that path. ([#2372](https://github.com/Revive-Home/revive-dashboard/pull/2372))
- **Intake address prefill** — Address details passed via URL parameters now prefill correctly, and the deal is created in the background so users can continue through the flow without waiting. ([#2363](https://github.com/Revive-Home/revive-dashboard/pull/2363))

### Action required

No action required for existing integrations. The new `email` and `phone` fields on `POST /v1/brokerages/` are optional and backwards compatible.

</Update>

<Update label="April 14, 2025" description="v2.154.0" tags={["revive-api"]}>

### New

- The estimate proposal response now includes a `homeIncrease` field in the `valueOpportunity` object. When present, it contains a formatted currency string (e.g. `"$45,000"`) representing the estimated increase in home value from renovations. ([#1953](https://github.com/Revive-Home/revive-api/pull/1953))

### Improved

- Estimate proposals and opportunity lookups load faster — comparables, case studies, HubSpot metadata, and associated records are now fetched concurrently. ([#1953](https://github.com/Revive-Home/revive-api/pull/1953))
- The `trendText` label in the comparison chart now reads `"$X home increase"` instead of `"$X est. profit"` to better describe the value. ([#1953](https://github.com/Revive-Home/revive-api/pull/1953))

### Fixed

No bug fixes in this release.

### Action required

- **`trendText` label changed** — The `trendText` field in the `comparisonChart` section of `GET /v1/estimates/{estimateId}/proposal` now uses "home increase" instead of "est. profit". If you parse or match against this string, update your code to expect the new label.

</Update>

<Update label="April 14, 2025" description="v2.153.0" tags={["revive-api"]}>

### New

- The Sell 360 cash offer comparison now subtracts existing mortgage/liens from the cash offer amount, giving homeowners a more accurate side-by-side comparison. ([#1951](https://github.com/Revive-Home/revive-api/pull/1951))

### Improved

- RMA contract generation now resolves the contractor signer from the completed PC contract instead of HubSpot contact associations, ensuring the signer on the RMA matches the signer who executed the PC contract. ([#1951](https://github.com/Revive-Home/revive-api/pull/1951))
- Generating an RMA contract once again requires a signed Purchase Commitment (PC) contract on the associated opportunity (restores the requirement removed in v2.151.0). ([#1951](https://github.com/Revive-Home/revive-api/pull/1951))

### Fixed

- The renovation plan form now correctly saves the **rehab used** amount when updating estimates. Previously, this value could be omitted from the save payload.

### Action required

- **Signed PC contract required for RMA generation** — The `POST /v1/rma/:rmaId/contract` endpoint now requires a completed PC contract on the associated opportunity. If no completed PC contract exists, the endpoint returns a `400` error. This reverses the v2.151.0 change. Ensure a PC contract is completed before generating RMA contracts.
- **Sell 360 `comparisonDiff` values will change** — The `comparisonDiff` field in `GET /v1/estimates/{estimateId}/proposal` now reflects a lower cash offer baseline (debt is subtracted). If you display or use `comparisonDiff` in calculations, values will be higher for homeowners with existing mortgage balances.

</Update>

<Update label="April 14, 2025" description="v2.152.0" tags={["revive-api"]}>

### New

- Tiered Sell 360 program fee — properties with ARV ≥ $1M now use a 4% rate; below $1M uses 3%. Previously a flat 3% applied to all. ([#1943](https://github.com/Revive-Home/revive-api/pull/1943))
- `debt` and `agentCommission` fields added to the `sell360TransactionSummary.shared` object so clients can display these deductions. ([#1943](https://github.com/Revive-Home/revive-api/pull/1943))
- New `firstCheckAmount` field on opportunity and deal responses, representing the first check amount stored in HubSpot. ([#1943](https://github.com/Revive-Home/revive-api/pull/1943))

### Improved

- Revised holding cost formula: `(initialPayment + renovationCost) × 1% × holdingMonths` — produces lower holding costs for most deals. ([#1943](https://github.com/Revive-Home/revive-api/pull/1943))
- Simplified taxes/insurance/HOA estimate: `asIsPrice × 0.16% × holdingMonths`. ([#1943](https://github.com/Revive-Home/revive-api/pull/1943))
- Break-even calculation no longer includes holding costs. ([#1943](https://github.com/Revive-Home/revive-api/pull/1943))
- Sell 360 comparison baseline now uses `asIsPrice × 90%` instead of a full as-is net calculation. ([#1943](https://github.com/Revive-Home/revive-api/pull/1943))
- Renovation timeline is now budget-based: 1 month per $100K increment, producing more consistent scenario projections. ([#1950](https://github.com/Revive-Home/revive-api/pull/1950))
- Initial payment now sourced directly from the opportunity's `firstCheckAmount` instead of a break-even formula. ([#1950](https://github.com/Revive-Home/revive-api/pull/1950))
- RMA contractor signer now sourced from the completed PC contract instead of HubSpot. ([#1943](https://github.com/Revive-Home/revive-api/pull/1943))

### Fixed

No bug fixes in this release.

### Action required

- **`agentCommissionSold` renamed to `firstTransactionCommission`** — Update any client code that reads this field on opportunity and deal responses.
- **`isFirstCheckNegative` removed** — This boolean has been removed from `sell360TransactionSummary`. Check `firstCheckNet` directly instead.
- **RMA contract generation requires completed PC contract** — `POST /v1/rma/:rmaId/contract` returns `400` if no completed PC contract exists.
- **Sell 360 scenario values will change** — All calculated values (initial payment, holding costs, program fees, scenario projections) may differ due to the formula changes above.

</Update>
