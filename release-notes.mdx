<Update label="Week of May 25, 2026" description="Weekly recap" tags={["revive-api", "revive-admin", "revive-dashboard"]}>

A skim-friendly recap of what shipped this week. See the dated entries below for full per-release detail.

### New

- **Public status page and health API.** A new `/status` page on the admin app and a `GET /api/health-status` endpoint surface the live operational state of HubSpot, Firebase, ATTOM (Estated), RestB, PandaDoc, AnyProp, and Revive's own API, dashboard, and admin app. The page auto-refreshes every minute and both endpoints can be embedded from any origin. See [Third-party status](/guides/health).
- **Background health checks with Slack alerts.** Health checks now run every 5 minutes in the background and post a Slack alert when a service transitions between `operational`, `degraded`, and `down` in production, so the team is notified before users feel it.
- **Total cost in admin estimate PDFs.** The scope and full estimate PDFs now show the total cost in the header alongside the property address, so reviewers see the headline number on every page.

### Improved

- **Cleaner notes formatting in estimate PDFs.** Rich-text styles pasted into estimate notes are stripped on PDF export, so notes render consistently and don't carry over stray fonts or colors from the source. Long property addresses also stay pinned to the address column without wrapping into other fields.

### Action required

None. All changes are additive or transparent to existing integrations.

</Update>

<Update label="Week of May 22, 2026" description="Weekly recap" tags={["revive-api", "revive-dashboard", "revive-admin"]}>

A skim-friendly recap of what shipped this week across the API, dashboard, and admin app. See the dated entries below for full per-release detail.

### New

- **Revive AI Report v4 is live end-to-end.** The next-generation report experience is now available across the homeowner dashboard and admin panel, with [AnyProp](/third-party-integrations/property-data) comparables, room-aware photo grouping, and a refreshed renovation estimate flow. Existing report URLs continue to work — v4 is served automatically.
- **Faster, mobile-friendly comparables analysis.** Comparable analysis runs noticeably faster, with grade badges on each comp, a tighter mobile layout, and smoother progress messaging while results load.
- **RenoVision and Comparables slides in the preliminary estimate.** The shareable Renovate-to-Stay pre-estimate now includes an AI-rendered "after" view and a focused comparable-homes slide alongside the renovation breakdown.
- **PDF download for preliminary estimates.** A one-click PDF export is available from the preliminary estimate view, with the property address used as the filename so saved files are easy to find and share.
- **Auto-filled RMA material advance.** Creating an RMA against a deal with a PO sheet now populates the material advance automatically — no more manual copying between sheets.
- **Renovate-to-Stay sharing without ARV / as-is values.** Admins can share a Renovate-to-Stay project before ARV and as-is numbers are filled in, removing a blocker when those figures aren't known yet.
- **Scope item labels in weekly updates.** Custom scope-item labels now appear in the weekly update scope picker and in the homeowner-facing view. Homeowners see the same item names project managers use. See [AI weekly updates](/guides/ai-weekly-updates).
- **RMA rush from the list view.** Any admin can request or remove rush on an RMA directly from the RMA list. Approve and decline still require the `rmaAdmin` permission.

### Improved

- **Live HubSpot pipeline stage on RMAs.** RMA detail views now read the current pipeline stage label directly from HubSpot, so the displayed stage always matches the latest configuration. See [HubSpot integration](/third-party-integrations/hubspot).
- **Mobile layouts across the report.** The Home Condition Summary and Mortgage section pick up dedicated mobile layouts, and the rejection modal now surfaces selling options inline.
- **Refined sharing and filtering on the report.** Share options use a single popover, and the report dashboard filter modal includes a popover-based date picker with cleaner service options.
- **Public access on shareable report routes.** Routes intended to be publicly shareable no longer require sign-in, so unauthenticated viewers can load them as designed.
- **Estimate display polish.** Pricing columns are wider for readability, and parent items now show a single `TBD` badge when every sub-item is TBD.
- **Owner's rep "Select All" persists.** The owner's rep filter on the admin RMA list keeps "Select All" across page navigation.
- **Address-based PDF filename in admin estimates.** Admin estimate PDFs now render from the correct view and use an address-based filename, so saved PDFs are easier to identify.

### Fixed

- **Editing Revive AI reports saves correctly.** Resolved a regression where some edits to a report were silently dropped on save, including the "Save changes" button in the edit-value modal.
- **RenoVision configurator stability.** Switching design packages no longer surfaces a stray "Apply Changes" prompt, changing one material no longer resets the others, and the configurator no longer gets stuck loading or re-renders in a loop.
- **Best-selling condition shown reliably.** The "best-selling condition" indicator on the report now renders consistently where it was previously missing in some scenarios.
- **Subject property persists after analysis.** The property entered during a Revive AI analysis is no longer cleared from the form once results come back.
- **Cleaner comparables analysis.** Concurrent refresh attempts collapse into a single request, status messages no longer loop, and duplicate completion chimes have been silenced.
- **Safer report rendering.** Additional null checks eliminate a class of rendering errors when partial data is loaded.
- **RMA check-run cutoff respects timezone.** RMAs submitted near the daily cutoff are now categorized into the correct week across timezones.
- **Cents rounding on monetary totals** eliminates sub-cent drift in derived figures.
- **Concierge ID resolution.** Corrected the Revive Concierge identifiers used in opportunity routing, so concierge assignments resolve to the right user.
- **More resilient external property data retries.** Rate-limit retry logic for upstream property-data calls is more robust, reducing transient failures during report generation.

### Action required

None. All changes are additive or transparent to existing integrations. See the dated entries below for per-release detail.

</Update>

<Update label="May 21, 2026" description="Revive AI Report v4" tags={["revive-api", "revive-dashboard", "revive-admin"]}>

### New

- **AnyProp as a property data and imagery provider.** The platform now sources comparable listings, property records, and home photos from AnyProp in addition to existing providers. AnyProp comps are normalized into the same shape as other sources, so reports, comparables, and renovation estimates work the same way regardless of where the underlying data came from.
- **`POST /v1/utility/get-home-images`.** A new endpoint returns home photos for an address. With no `source` specified it falls back across providers (RestB → AnyProp → Zillow) and returns the first successful, deduplicated set. Pass `source: "restb" | "anyprop" | "zillow"` to pin a specific provider. Used during intake to confirm a property has usable imagery before kicking off a report.
- **`POST /v1/utility/deduplicate-images`.** A new endpoint accepts an array of image URLs and returns the set with exact and visually-similar duplicates removed via perceptual hashing. Useful when combining photos from multiple sources before sending them into a render or report.
- **Room-type classification on comparable photos.** Comparable images returned by report endpoints now include per-image room types (kitchen, bathroom, bedroom, living, exterior, etc.) and an aggregated `roomTypes` list per comparable. Results are cached after the first request, so reopening a comp is instant.

### Improved

- **Revive AI Report v4.** Reports now include AnyProp-sourced comparables, room-aware photo grouping, and the updated renovation estimate flow end-to-end across the homeowner dashboard and admin tools. Existing report URLs continue to work — v4 is served automatically.
- **Meeting scheduling polish.** The Getting Started meeting scheduler has tighter dropdown behavior, cleaner advisor selection, and fewer redundant components in the booking flow.

### Fixed

- **Vercel build stability.** Resolved module-resolution issues that intermittently broke dashboard and admin deployments on Vercel.

### Action required

No action required. The new `/v1/utility/*` endpoints are additive, and the new `source` and room-type fields on existing responses are additive. Existing integrations continue to work unchanged.
