## Prerequisites

Before you begin, make sure you have:

- **Node.js** v18+ and **npm** v9+
- **Git** with access to the [Revive-Home](https://github.com/Revive-Home) GitHub org
- **Firebase CLI** — `npm i -g firebase-tools`
- A `.env` file (ask the team lead for the template)

---

## Repository setup

Clone the repo you're working on:

```bash
git clone git@github.com:Revive-Home/revive-api.git
git clone git@github.com:Revive-Home/revive-dashboard.git
git clone git@github.com:Revive-Home/revive-admin.git
git clone git@github.com:Revive-Home/revive-mobile.git
```

### revive-api (backend)

```bash
cd revive-api
npm install
cp .env.example .env    # fill in Firebase, HubSpot, Stripe keys
npm run dev              # starts on http://localhost:3001
```

The API runs on port **3001** by default. Health check: `curl http://localhost:3001/health`

### revive-dashboard (web app)

```bash
cd revive-dashboard
npm install
cp .env.example .env.local
npm run dev              # starts on http://localhost:3000
```

### revive-admin (internal tools)

```bash
cd revive-admin
npm install
cp .env.example .env.local
npm run dev              # starts on http://localhost:3002
```

### revive-mobile (React Native)

```bash
cd revive-mobile
npm install
npx pod-install          # iOS only
npm run ios              # or npm run android
```

## Environment variables

Each repo uses a `.env` or `.env.local` file. Key variables shared across repos:

| Variable | Used by | Description |
|----------|---------|-------------|
| `FIREBASE_PROJECT_ID` | All | Firebase project for auth and Firestore |
| `FIREBASE_API_KEY` | Dashboard, Admin, Mobile | Client-side Firebase config |
| `FIREBASE_SERVICE_ACCOUNT` | API | Server-side Firebase admin SDK |
| `HUBSPOT_API_KEY` | API | HubSpot CRM integration |
| `PANDADOC_API_KEY` | API | PandaDoc e-signature integration |
| `STRIPE_SECRET_KEY` | API | Stripe payment processing |
| `DATABASE_URL` | API | PostgreSQL connection string |

<Warning>
  Never commit `.env` files. They are gitignored in all repos. Ask your team lead for the current values.
</Warning>

## Running the full stack

For end-to-end development, you'll typically run:

| Service | Command | Port |
|---------|---------|------|
| `revive-api` | `npm run dev` | 3001 |
| `revive-dashboard` | `npm run dev` | 3000 |
| `revive-admin` | `npm run dev` | 3002 |

The dashboard and admin connect to the API at `http://localhost:3001` by default (configured in `.env.local`).

## Common tasks

### Run tests

```bash
npm test                 # unit tests
npm run test:e2e         # end-to-end tests (API only)
```

### Database migrations

```bash
npm run migrate          # run pending migrations
npm run migrate:undo     # rollback last migration
```

### Generate release notes

```bash
# From revive-api-docs repo
RELEASE_VERSION=v2.155.0 RELEASE_SINCE=2026-04-01 node scripts/generate-release-notes.mjs
```

### Preview these docs locally

```bash
cd revive-api-docs
npm i -g mint
mint dev                 # http://localhost:3000
```

## Branch conventions

| Branch | Purpose |
|--------|---------|
| `main` | Production — auto-deploys |
| `staging` | Staging environment |
| `feature/*` | Feature branches — PR into `main` |
| `fix/*` | Bug fix branches — PR into `main` |

All PRs require at least one review. Merging to `main` triggers a production deployment.
