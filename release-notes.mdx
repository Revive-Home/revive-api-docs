<Update label="April 23, 2026" description="v2.133.0" tags={["revive-admin"]}>

### New

- **RMA list — deal type column and pipeline stage.** The RMA list now shows the opportunity's deal type (for example, Renovate to sell, Sell 360, Flip 360, Renovate to stay) alongside pipeline stage, making it easier to triage RMAs across programs. Deal type labels are normalized from the HubSpot `dealtype` values. ([#1755](https://github.com/Revive-Home/revive-admin/pull/1755))
- **RMA `executed_at` timestamp.** RMAs now record the moment they move to the Executed status. The RMA detail page shows "Executed on M/D/YY at H:MM AM" beneath the status badge. ([#1758](https://github.com/Revive-Home/revive-admin/pull/1758))
- **Custom items support TBD pricing in RET.** When pricing isn't final, you can flag a custom line item as **TBD** in the Renovation Estimate Tool. TBD items appear in the estimate preview and review panel without a firm price, and the flag is reset automatically if the item is converted to a non-custom product. ([#1753](https://github.com/Revive-Home/revive-admin/pull/1753))
- **Group dropdown on custom items.** Custom items in RET now include a **Group** selector in both the custom item modal and the review side panel, so custom work can be grouped alongside catalog products for cleaner proposals. ([#1756](https://github.com/Revive-Home/revive-admin/pull/1756))
- **Groups on Level 1 products in the RET Catalog.** Catalog admins can now assign a group to Level 1 (global) products when adding or editing catalog items. Groups flow through to Level 2 and Level 4 drag-and-drop reordering. ([#1752](https://github.com/Revive-Home/revive-admin/pull/1752))

### Improved

- **Check Run tabs use execution date.** The **Ready for Check Run** and **Next Check Run** tabs now filter by `executed_at` (falling back to `updated_at`) instead of `created_at`, so RMAs appear in the correct tab based on when they were actually executed. ([#1758](https://github.com/Revive-Home/revive-admin/pull/1758))
- **Check run cutoff moved to Friday.** The weekly cutoff for the Ready list shifted from Thursday to Friday 11:59 PM. RMAs executed on Friday now stay in **Next Check Run** until Saturday, keeping the Ready list stable on processing day. ([#1758](https://github.com/Revive-Home/revive-admin/pull/1758))

### Fixed

- Fixed a TypeScript error in Level 2 and Level 4 drag-and-drop reorder by including the missing `group` field on catalog product updates. ([#1752](https://github.com/Revive-Home/revive-admin/pull/1752))
- The **TBD** flag on custom items is now reset when an item is changed from custom to a catalog product, preventing stale TBD state in the RET preview. ([#1753](https://github.com/Revive-Home/revive-admin/pull/1753))

### Action required

No action required for existing integrations. These changes are internal to Revive Admin and do not affect the public API.

</Update>

<Update label="April 23, 2026" description="Dashboard v1.150.0" tags={["revive-dashboard"]}>

### New

- Agents can add a new brokerage directly from the **My profile** screen in account settings. The new **Add brokerage** modal collects the brokerage name, website, logo, optional street address, and optional email and phone. Brand colors are auto-extracted from the uploaded logo. On success, the new brokerage is selected on the profile automatically.
- Custom estimate items can now display a **TBD** label when no price has been set. In the proposal view, items flagged as TBD render a red `TBD` badge in place of the total cost, signalling that the price is to be determined rather than hiding the item. This is driven by an `isTBD` flag on custom estimate items.

### Improved

- Disclaimer copy across pre-estimate slides has been updated for clarity and legal accuracy:
  - **Nearby case studies** slide now includes: _"Individual results may vary. Testimonials are not intended to guarantee the same or similar results."_
  - **Value opportunity** slide now includes: _"All projected costs, values, and potential outcomes are estimates only and may vary based on market conditions, property-specific factors, and final project scope."_
  - **Sell360 transaction summary** and **scenario comparison** disclaimers have been consolidated into shorter, clearer statements covering estimated figures, timing of payments, and non-binding proposal language. The scenario comparison PDF export reflects the same updated copy.
- The **Save changes** button on **My profile** is disabled immediately after creating a new brokerage to prevent duplicate submissions while the profile refreshes.
- Stale brokerage search responses are now aborted when the query changes, so the dropdown always reflects the latest input.

### Fixed

- **Intake "What help" screen** — Removed the `refer-renovate` flow branch that caused the "What help do you need?" screen to be skipped in certain intake funnels.
- **Custom item TBD label position** — The TBD label now renders inside the item's content area so it aligns with other price elements instead of appearing detached above them.

### Action required

No action required for existing integrations. The `POST /v1/brokerages/` request body continues to accept the fields documented in the [API Reference](/api-reference) — `address`, `city`, `state`, and `zip` are optional, and `email` and `phone` are accepted optionally when creating a brokerage from the dashboard.

</Update>

Check run totals are now computed from the full tab-filtered set, not just the current page. ([#1758](https://github.com/Revive-Home/revive-admin/pull/1758))
- **Catalog access** — `carlo@revive.realestate` and `nehme@revive.realestate` added to the RET Catalog allowed-email list. ([#1756](https://github.com/Revive-Home/revive-admin/pull/1756))

- **`isTBD` reset on non-custom items** — Non-custom RET items no longer retain an `isTBD` flag when edited through the preview panel. ([#1753](https://github.com/Revive-Home/revive-admin/pull/1753))

### Action required

- **Catalog items require a `group`** — Existing L1 catalog items without a `group` (or with a group not in the current list) will show an empty **Group** dropdown and must be updated before saving. Keep the admin list in sync with `revive-api`'s `ret-catalog.validation.js` `.valid()` list.
- **Custom items require a `group`** — Saving a custom item without selecting a group now fails validation. Existing custom items on in-flight estimates should be re-opened and assigned a group.
- **RMA `dealtype` field is now a label** — `useRmas` maps `opportunity.dealtype` through `getDealTypeLabel` before returning it, so consumers see `Renovate to sell`, `Sell 360`, etc. instead of raw HubSpot IDs like `Contract Buy-out` or `Buy It Now`. If you filter on this field client-side, use the label values.
- **RMA check-run filter uses `executed_at`** — The Ready and Next Check Run tab split now uses `executed_at || updated_at` and a Friday cutoff. RMAs executed on Friday itself roll into Next Check Run until Saturday.

</Update>
